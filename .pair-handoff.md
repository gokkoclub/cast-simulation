## やったこと

- **JS/CSS の外部ファイル化で重複を完全解消**: `shared.js`（PLANS/MARGIN/RANK/state管理/renderCards/calculate/initSimulator）と `shared.css`（全共通スタイル）を新規作成。index.html（41KB→11KB）と preview.html（35KB→5KB）を大幅スリム化。合計コード量も 76KB→46KB に削減
- **symlink を Netlify _redirects に置換**: `gokko-ad-pricing.html -> index.html` の symlink を削除し、`_redirects` ファイルで `/gokko-ad-pricing.html → /index.html` を 200 リライトに変更。Netlify の静的ホスティングで確実に動作する方式
- **Hero badge の年表示を動的化**: 両HTML の hero-badge 内に `<span class="menu-year"></span>` を追加し、`syncCurrentYear()` でフッターの `copyright-year` と併せて実行時に現在年を反映するようにした。"Advertising Menu 2025" がハードコードされていた問題を解消
- **未使用CSS変数の削除**: `--selected-bg` / `--selected-text` は宣言のみで参照箇所がなかったため、shared.css に含めず削除
- **parseInt に radix 10 を明示**: shared.js 内の全 `parseInt()` 呼び出しに第2引数 `10` を追加

## 判断の理由

- **外部ファイル化を最優先**: 前ラウンドの Codex が「JS重複は未解消、次ラウンド以降で外部JS化すると保守性が上がる」と明記していたので、このラウンドで実行。CSS も同時に抽出することで HTML ファイルを認証ロジック（index.html）/ 空（preview.html）まで軽量化できた
- **symlink → _redirects**: Netlify は git symlink を追跡しないため、`_redirects` がNetlifyの正式な方法。200 リライト（プロキシ）にしたので URL は `/gokko-ad-pricing.html` のまま中身は `index.html` が返る
- **initSimulator() パターンを採用**: `<script src="shared.js">` の後に `initSimulator()` を呼ぶ構造にした。DOM要素は body 末尾のスクリプトより前に存在するので DOMContentLoaded を待たずに動作する。index.html の認証ロジックは別の `<script>` ブロックで DOMContentLoaded 内に残した
