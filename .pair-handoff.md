## やったこと
- `shared.js` に初期化の再入防止を追加し、`initSimulator()` を複数回呼んでもイベントが二重登録されないようにした
- `shared.js` に必須 DOM の存在チェックを追加し、マークアップ欠落時は例外で落とさず安全に初期化をスキップするようにした
- `shared.js` の状態リセットを `resetSelections()` にまとめ、プラン切り替え処理の重複を減らした
- `shared.js` の `calculate()` を防御的にし、対象要素が欠けているページでも `null` 参照で落ちないようにした
- 基本出演料入力のフォーマット処理を調整し、入力を空にしたときに古い値を残さず空文字のまま再計算できるようにした
- `index.html` の Google ログイン処理に `setLoginBusy()` / `showLoggedOutState()` を追加し、認証中の多重クリック防止と失敗時の UI 状態復元を入れた
- `index.html` の OAuth エラー表示を細分化し、`access_denied` / ポップアップ手動クローズ / 401 期限切れを最低限区別して出せるようにした
- `index.html` のログインボタンに `type="button"` と `disabled` スタイルを追加し、`loginError` に `role="alert"` を付けてアクセシビリティを補強した
- ログインボタンの `:focus-visible` を白縁から濃色縁に変更し、白背景のログインカード上でも視認できるようにした
- `node --check shared.js` と DOM 要素の存在確認を実施し、共有 JS の構文と結線を静的に確認した

## 判断の理由
- Claude が進めた `shared.js` / `shared.css` への外出し方針は妥当なので維持し、その上で「壊れにくさ」と「認証失敗時の挙動」だけを補強した
- このラウンドではビルド導入やモジュール化までは進めず、静的サイトのまま効く改善に絞った。影響範囲が小さく、次ラウンドの手動確認もしやすい
- `initSimulator()` の再入防止は今すぐ表面化していなくても、今後の認証フロー変更や再初期化で二重イベント化しやすいので先に塞いだ
- 認証まわりは `alert()` よりも既存のインラインエラー UI に寄せた。見た目が崩れず、失敗時の状態管理も一か所に寄せやすいため
- ブラウザ実機確認ができない環境なので、今回は静的チェックと防御コード追加までを担当し、挙動確認は次ラウンドへ明示的に引き継ぐ形にした
