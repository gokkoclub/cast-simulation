## やったこと
- `shared.js` の `selectPlan()` にプラン番号のバリデーションを追加し、`data-plan` が壊れていても不正な `currentPlan` が入らないようにした
- `shared.js` の `getCoeffCategories()` を防御的にし、`currentPlan` が不正でも `PLANS[0]` にフォールバックするようにした
- 係数カードのラジオ系オプション（`coeff-option`）に roving tabindex を追加し、選択中の項目だけ `tabindex="0"`、非選択項目は `-1` になるようにした
- 係数カードのラジオ系オプションに Arrow Left/Right/Up/Down と Home/End のキーナビゲーションを追加し、WAI-ARIA の radiogroup 操作に寄せた
- `shared.js` の `handleClick()` にインデックス妥当性チェックを追加し、壊れた `data-idx` や将来の不整合で落ちないようにした
- `index.html` と `gokko-ad-pricing-preview.html` の静的 HTML でも、非アクティブなプランボタンを `tabindex="-1"` に修正して初期マークアップの整合性を合わせた
- `index.html` の OAuth callback で `access_token` も `error` もない場合に、busy 状態解除だけでなく汎用エラー文言も出すようにした
- `node --check shared.js` で構文チェック通過を確認した

## 判断の理由
- Claude の IIFE 化、OAuth `error_callback` 追加、プランセレクターの roving tabindex は妥当だったため維持し、その上で未完了だった係数カード側のアクセシビリティを補った
- 係数カードは動的再描画されるので、個別 DOM を都度直接操作するより、`renderCards()` で `tabindex` を再計算する方が状態不整合を起こしにくい
- チェックボックス群（`media-option`）まで矢印キー移動を広げることもできるが、まずは handoff で明示されていた radiogroup 側を優先した。チェックボックス群は Tab と Space で操作可能なため、優先度を下げた
- HTML の初期 `tabindex` も合わせて直したのは、JS 実行前のアクセシビリティツリーと実行後の挙動を一致させるため。JS 初期化だけに依存しない方が安全
