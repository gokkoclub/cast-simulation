## やったこと
- `shared.js` を IIFE で包んでグローバルスコープ汚染を解消。外部に公開するのは `window.initSimulator` のみ（`PLANS`, `MARGIN`, `RANK`, `state` 等はすべてクロージャ内に閉じ込め）
- `shared.js` の `calculate()` にインデックス境界チェックを追加。`state.period` 等が範囲外の場合にフォールバック（`|| plan.period[0]`）で crash を防止。`state.media` の各要素も存在確認してから加算するよう修正
- プランセレクター（`role="radiogroup"`）に WAI-ARIA 準拠の roving tabindex + 矢印キーナビゲーションを実装。Arrow Left/Right/Up/Down で隣接プランへ移動、Home/End で先頭/末尾へジャンプ。`selectPlan()` でも非アクティブボタンの `tabindex` を `-1` に設定
- `initSimulator()` 内で HTML の tabindex を JS で初期化時に補正（`active` ボタンのみ `tabindex="0"`）
- `index.html` の Google OAuth `initTokenClient` に `error_callback` を追加。ポップアップブロック（`popup_failed_to_open`）・ポップアップ手動クローズ（`popup_closed`）・その他エラーを区別して表示し、`setLoginBusy(false)` でボタンを復帰
- `index.html` の OAuth callback で `access_token` も `error` もない異常レスポンス時に `setLoginBusy(false)` を呼ぶ else 節を追加（ボタン固まり防止）
- `node --check shared.js` で構文チェック通過を確認

## 判断の理由
- IIFE 化は Codex が前ラウンドで「次回は IIFE か ES Modules 化を検討してよい」と引き継いでいたので実施。ES Modules はビルドステップや `type="module"` への変更が必要で影響が大きいため、IIFE を選択
- OAuth `error_callback` は実際のバグ（ポップアップブロック時にボタンが disabled のまま永久に固まる）の修正。Google Identity Services のドキュメントで `error_callback` がインフラレベルのエラー（callback が呼ばれない場合）を処理するために存在することを確認
- roving tabindex は WAI-ARIA radiogroup パターンの要件。既に `role="radiogroup"` を宣言している以上、矢印キーでの操作が期待される。これがないとスクリーンリーダーユーザーが正しく操作できない
- 境界チェックは現状の正常フローでは到達しないが、将来のプラン数変更やバグによる state 不整合時の安全弁として追加。コストがほぼゼロなので入れる判断

## 次のパートナーへ
- **ブラウザ実機確認が未実施**（3ラウンド通じて静的チェックのみ）。最低限以下を確認すべき:
  - `gokko-ad-pricing-preview.html`: プラン切り替え・係数変更・入力クリア・矢印キーでのプラン移動が正常に動くか
  - `index.html`: Google ログイン成功→overlay 消え→アプリ表示、ログイン失敗→エラー表示+ボタン復帰、ポップアップブロック時のエラー表示
  - Hero badge と footer の年表示が動的に正しい年（2026）になっているか
  - Netlify 実デプロイで `_redirects` が機能するか
- **係数カード（coeff-option）の radiogroup にも roving tabindex + 矢印キーを追加すべき**。今回はプランセレクターのみ実装。係数カードは動的再生成されるため、renderCards 内での実装が必要で影響範囲が大きいため見送り
- **HTML テンプレートの重複**は引き続き残存（`index.html` と `gokko-ad-pricing-preview.html` のシミュレーター部分がほぼ同一）。共通化するなら SSI、テンプレートエンジン、または Web Components の導入が必要
- **トークン有効期限の処理**が未実装。Google アクセストークンは通常1時間で失効するが、ログイン後に長時間操作した場合のリフレッシュや再認証プロンプトがない
- **`'use strict'`** は IIFE 内でのみ有効。`index.html` のインライン `<script>` 部分はまだ sloppy mode のまま
