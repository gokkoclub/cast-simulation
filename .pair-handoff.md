## やったこと

- **認証エラー表示の整理**: `index.html` の Google ログイン処理に `showLoginError()` / `hideLoginError()` を追加し、OAuth コールバックエラー・許可外ドメイン・トークン検証失敗でメッセージを出し分けるようにした。失敗時は `sessionStorage` のトークンも明示的に削除
- **アクセシビリティの追加改善**: `index.html` / `gokko-ad-pricing-preview.html` の両方で `:focus-visible` のフォーカスリング、基本出演料入力の `aria-label` と `inputmode="numeric"`、結果読み上げ用の `aria-live` リージョン（`#resultLiveRegion`）を追加
- **係数カードのセマンティクス補強**: 各係数グループに `role="radiogroup"` または `role="group"` を付与。固定のベース媒体（TikTok）は `aria-disabled="true"` と `tabindex="-1"` にして、見た目は残しつつ操作対象から外した
- **年表示の動的化**: フッターの `© 2025` を固定値のまま残さず、両HTMLで実行時に現在年を反映する `copyright-year` に変更
- **軽いコード整理**: `index.html` の未使用 `cachedAccessToken` と、両HTMLの未使用ローカル変数を削除してノイズを減らした

## 判断の理由

- **Claude の認証方針は維持**: `sessionStorage` の存在だけで通していた旧実装には戻さず、Claude が入れた「トークンを毎回 Google `userinfo` で再検証する」流れはそのまま活かした。その上で、失敗ケースのUIと状態リセットだけ補強した
- **a11y は小さく確実に積んだ**: 大きなUI構造変更やJS分割はこのラウンドでは触らず、既存DOMを壊さない `aria-live` / `focus-visible` / `inputmode` / グループロール追加に絞った。差分を小さく保てて、preview版にも横展開しやすい
- **年表示は動的化を優先**: 2026への単純置換だと来年また同じ修正が必要になるため、固定値更新ではなく実行時に現在年を入れる方式にした
- **symlink は今回は維持**: `gokko-ad-pricing.html -> index.html` は重複削減として合理的なので戻していない。Netlify上での最終挙動はこの環境から確認できないため、実デプロイ確認タスクとして残した

## 次のパートナーへ

- **ブラウザでの手動確認が未実施**: この環境では実ブラウザ確認をしていない。少なくとも `@gokkoclub.jp` 正常ログイン、許可外ドメイン拒否、期限切れ/無効トークンでのセッション復帰失敗、preview版のキーボード操作を確認してほしい
- **Netlifyとsymlink確認**: `gokko-ad-pricing.html` の symlink を Netlify がそのまま配信できるかは未検証。もし崩れるなら `gokko-ad-pricing.html` を実ファイルへ戻すか、ビルド時コピーに切り替える必要がある
- **JS重複は未解消**: `index.html` と `gokko-ad-pricing-preview.html` の `PLANS` / `renderCards()` / `calculate()` などはまだ重複している。次ラウンド以降で外部JS化すると保守性が上がる
- **認証メッセージの文言改善余地**: 今回は最低限の出し分けだけ追加した。必要なら「ネットワーク失敗」「期限切れ」「キャンセル」などでもう少し具体的な文言にしてよい
